## 🧒 한마디로 이게 뭐예요?

> **증권사 전문가들이 "이 주식 살까 말까"를 분석한 보고서를,
> AI가 읽고 주제별로 정리해 쌓아두는 "주식 위키백과"예요.**

증권회사에는 **애널리스트**라는 주식 분석 전문가들이 있어요.
이들은 매일 "삼성전자 목표주가 60만원!", "지금은 사지 마세요" 같은
**리포트(보고서)** 를 써요.

그런데 이런 보고서가 **수백 개**가 쌓이면 다시 찾아 읽기가 너무 힘들어요.
그래서 이 프로젝트는 보고서를 읽을 때마다 핵심만 뽑아
**"삼성전자" 노트 한 장에 계속 쌓아 둬요.**

나중엔 그 노트 한 장만 보면 *"여러 전문가가 삼성전자를 어떻게 봤는지"* 가
한눈에 보여요. 시험 때 교과서 대신 **요약 노트**를 보는 것과 똑같은 아이디어죠. 📒

## 🔗 마법의 연결고리: `[[대괄호]]`

노트 안에 `[[삼성전자]]` 처럼 **대괄호 두 개**로 감싼 단어가 보이면,
그건 **클릭하면 그 페이지로 점프하는 링크**예요.
위키백과나 게임 속 하이퍼링크랑 똑같죠.

이렇게 회사·전문가·테마가 서로 연결되면서, 사이트에 들어가면
**거미줄 같은 "관계 지도(그래프)"** 가 자동으로 그려져요. 🕸️

## 🤖 누가, 어떻게 노트를 쌓을까?

사람이 일일이 정리하지 않아요. **AI(Claude)** 가 합니다.

```
📰 새 보고서·뉴스   →   🤖 AI가 핵심 요약   →   📒 해당 노트에 추가/수정   →   ✅ 자동 저장
```

- AI는 `CLAUDE.md`의 **규칙**에 맞춰 노트를 만들고 업데이트해요.
- 목표주가가 바뀌면 **덮어쓰지 않고** 표에 새 줄을 추가해요. (과거 기록도 다 남겨요)
- 두 전문가 의견이 **충돌하면** "상충하는 견해" 칸에 둘 다 적어둬요. (한쪽을 지우지 않음)
- 실제로 이 위키는 `auto-ingest: proposal #111` 같은 기록처럼 **수백 번** 자동으로 채워졌어요.

> 💡 핵심 원칙: **정보를 지우지 않는다.** 오래된 의견도 날짜와 함께 보존하고 "obsolete(폐기됨)"라고 표시만 해요.

## 🌍 노트가 어떻게 진짜 웹사이트가 되나요?

우리가 쓴 건 그냥 **글자 파일(마크다운)** 이에요.
이걸 **Quartz** 라는 도구가 읽어서 예쁜 웹사이트로 바꿔주고,
**GitHub Pages** 가 인터넷에 띄워줘요.

```
📒 노트 쓰기        🛠️ 정리(stage)       ✨ Quartz 변환        🌐 인터넷 공개
wiki/ + index.md  ──▶  content/ 폴더  ──▶   public/ 폴더   ──▶  GitHub Pages
 (우리가 쓴 원본)      (빌드용 복사본)      (완성된 웹사이트)      (누구나 열람)
```

1. `stage.mjs` 가 `wiki/`·`index.md`·`raw/` 를 `content/` 폴더로 복사해요. (원본은 그대로 둬서 **단일 소스 유지**)
2. **Quartz** 가 `content/` 를 읽어 디자인·검색·그래프를 입힌 웹사이트(`public/`)를 만들어요.
3. `main` 브랜치에 올리면(push) GitHub가 **자동으로** 위 과정을 실행해 사이트를 새로고침해요. (`.github/workflows/deploy.yml`)

> 손글씨를 넣으면 인쇄된 책으로 만들어주는 기계처럼, **글만 쓰면** 멋진 사이트가 나와요. 🖨️

## ⚠️ 딱 하나 조심할 점 (가장 흔한 오류)

모든 노트의 맨 위에는 `---` 로 감싼 **"이름표"(프론트매터)** 가 있어요.
여기에 종목 이름·번호·섹터를 정해진 형식(**YAML**)으로 적어야 해요.

```yaml
---
type: stock
ticker: '005930'
name: 삼성전자
sector: "[[반도체]]"      # ← [[ ]] 가 들어가면 꼭 따옴표로 감싸기!
created: 2026-06-01
updated: 2026-06-03
---
```

형식이 조금만 틀려도 **웹사이트 만들기가 통째로 실패**해요. 규칙은 간단해요:
- 위아래를 `---` 로 정확히 감쌀 것 (백틱 ` ``` ` 아님!)
- `[[위키링크]]` 가 들어간 값은 **따옴표 `" "`** 로 감싸기
- 값이 여러 개면 콤마로 나열하지 말고 **목록(블록 시퀀스)** 으로 쓰기:
  ```yaml
  sector:
    - "[[자동차]]"
    - "[[2차전지]]"
  ```

## 📥 새 보고서를 추가하려면 (Ingest)

자세한 절차는 [`수집_가이드.md`](수집_가이드.md) 참고.

```bash
# 1) raw/reports/ 에 PDF 저장 — 파일명 규칙: YYYY-MM-DD_증권사_종목.pdf
#    예) 2026-05-07_미래에셋_SK하이닉스.pdf
# 2) PDF를 마크다운으로 변환
python pdf_to_md.py            # → raw/reports_md/*.md (ingested:false 로 표시됨)
# 3) Claude Code에게 부탁: "raw/reports_md/ 의 미반영 리포트를 ingest해줘"
#    → AI가 표에 줄 추가, 충돌 의견 정리, ingested:true 로 변경
```

> ⚠️ 본인 참고용 **수동 수집은 정상**이지만, 집계 사이트를 **대량 자동 스크래핑**하는 건 약관 위반이니 하지 마세요.
> 또한 외국계 IB 원문 PDF는 비공개라, 외국계 리포트는 **공개 보도 기반 2차 요약**이에요. (각 파일 이름표에 출처를 표시)
