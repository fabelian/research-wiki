현재 커버리지: **메모리 반도체 3강** (SK하이닉스 · 삼성전자 · 마이크론) 중심의 AI 메모리 슈퍼사이클.

## 🧒 이게 대체 뭐예요? (중학생도 이해하는 5분 설명)

**한 줄 요약:** 증권사 전문가들이 "이 주식 살까 말까"를 분석해 쓴 보고서들을,
**AI가 주제별로 깔끔하게 정리해 모아둔 나만의 위키백과**예요.

### 왜 만들었을까?
주식회사(예: 삼성전자)에 대해 증권사 애널리스트(전문 분석가)들은 매일 보고서를 써요.
"목표주가 10만원!", "지금은 사지 마세요" 같은 의견이죠.
그런데 이런 보고서가 **수십, 수백 개**가 되면 일일이 다시 읽기가 너무 힘들어요.

그래서 이 프로젝트는 보고서를 읽을 때마다 핵심만 뽑아서
**"삼성전자" 노트 한 장에 계속 쌓아 둬요.** 나중엔 그 노트 한 장만 보면
"여러 전문가가 삼성전자를 어떻게 봤는지"가 한눈에 정리돼 있는 거죠.
교과서를 다시 펴지 않아도 되도록 **나만의 요약 노트**를 만드는 것과 똑같아요.

### 폴더를 학교 사물함에 비유하면
| 폴더 | 비유 | 하는 일 |
|------|------|--------|
| `raw/` | 📦 원본 보관함 | 받은 보고서 원본. **절대 안 고쳐요** (증거 사진 찍어두듯) |
| `wiki/` | 📒 정리 노트 | 원본에서 핵심만 뽑아 주제별로 정리한 노트. AI가 관리 |
| `index.md` | 📑 목차 | 어떤 노트가 있는지 보여주는 첫 페이지 |
| `log.md` | 📅 일기장 | "오늘 무슨 작업을 했다"를 날짜순으로 기록 |
| `CLAUDE.md` | 📜 규칙표 | AI가 노트를 정리할 때 지켜야 할 약속 |

### 누가 정리하나요? → **AI가요!**
사람이 일일이 정리하는 게 아니라, **AI(Claude)** 가 보고서를 읽고
규칙(`CLAUDE.md`)에 맞춰 노트를 자동으로 만들고 업데이트해요.
사람은 그냥 **읽기만** 하면 돼요.

### 마법의 연결고리: `[[대괄호]]`
노트 안에 `[[삼성전자]]`처럼 대괄호 두 개로 감싼 단어가 있으면,
그건 **클릭하면 그 페이지로 점프하는 링크**예요. 위키백과나
게임 안의 하이퍼링크랑 똑같죠. 이렇게 종목·전문가·테마가
거미줄처럼 서로 연결돼서 "관계 지도(그래프)"가 만들어져요.

### 노트가 어떻게 진짜 웹사이트가 되나요?
우리가 쓴 노트(글자 파일)를 **Quartz**라는 도구가 읽어서
예쁜 웹사이트로 바꿔주고, **GitHub Pages**가 그걸 인터넷에 띄워줘요.

```
📒 노트 쓰기  ──▶  🛠️ Quartz가 웹사이트로 변환  ──▶  🌐 인터넷에 공개
 (wiki/ 폴더)         (디자인·검색·그래프 추가)        (누구나 열람)
```

> 💡 그래서 우리는 **글(노트)만 쓰면** 되고, 멋진 디자인·검색·다크모드는
> Quartz가 알아서 입혀줘요. 마치 손글씨를 넣으면 인쇄된 책으로 만들어주는 기계처럼요.

### ⚠️ 딱 하나 조심할 점
노트의 맨 위에는 `---` 로 감싼 **"이름표"(프론트매터)** 가 있어요.
여기에 종목 이름·번호·섹터 같은 정보를 정해진 형식(YAML)으로 적어야 하는데,
형식이 조금만 틀려도 웹사이트 만들기가 **실패**해요.
(예: `[[ ]]` 가 들어간 값은 꼭 따옴표 `" "` 로 감싸야 해요.)

## 🗂️ 구조 (3계층)

| 경로 | 역할 |
|------|------|
| `raw/reports/` | 원본 리서치(불변). `YYYY-MM-DD_증권사_종목.md` |
| `wiki/` | LLM이 소유·갱신하는 컴파일된 지식 (stocks · sectors · analysts · themes) |
| `CLAUDE.md` | 위키 유지·관리 계약(스키마) |
| `index.md` | 카테고리별 네비게이션 인덱스 (사이트 홈) |
| `log.md` | 시간순 append-only 작업 기록 |

**현재 페이지**: 종목 4 · 섹터 1 · 애널리스트 4 · 테마 4 / **원본 리포트 18건**

## 🚀 배포 작동 방식

작성 원본(`wiki/` + `index.md`)은 그대로 두고, 빌드 시점에 Quartz 입력으로 복사해 정적 사이트를 생성합니다 — **단일 소스 유지**.

```
wiki/ + index.md   ──(stage.mjs)──▶   content/   ──(Quartz)──▶   public/   ──▶   GitHub Pages
 (작성 원본)                          (빌드 입력)      (정적 사이트)        (배포)
```

- `main`에 push → `.github/workflows/deploy.yml`가 stage → build → 배포 자동 실행
- `content/`는 빌드 산출물(git 미추적), `public/`·`node_modules/`는 `.gitignore`

### 로컬에서 미리보기

```bash
npm install                          # 최초 1회
node stage.mjs && npx quartz build --serve   # http://localhost:8080
```

자세한 절차는 [`수집_가이드.md`](수집_가이드.md) 참조.

```bash
# 1) raw/reports/ 에 PDF 저장 (YYYY-MM-DD_증권사_종목.pdf)

python pdf_to_md.py            # → raw/reports_md/*.md (ingested:false)
# 3) Claude Code에서: "ingest raw/reports_md/ 의 미반영 리포트"
```

> 본인 참고용 수동 수집은 정상. 집계 사이트 대량 자동 스크래핑은 약관 위반이니 지양.
