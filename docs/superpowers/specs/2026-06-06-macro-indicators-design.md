# 거시지표 페이지 타입 추가 — 설계 문서

- 작성일: 2026-06-06
- 대상 위키: 셀사이드 리서치 위키 (`research-wiki`)
- 상태: 승인됨 (사용자 확정 2026-06-06)

## 1. 배경 / 목적

기존 위키는 종목·섹터·애널리스트·테마 4개 페이지 타입으로 셀사이드 리서치의 *해석*을 누적한다.
여기에 거시경제 지표(환율·유가·VIX·CNN 공포탐욕지수·한국/미국 금리·한국 CDS 스프레드)를 추가해,
리포트가 작성된 시장 국면을 함께 기록하고 거시 변수와 종목/섹터의 영향 관계를 구조화한다.

사용자 결정 요약:
- 거시 페이지를 독립적으로 두되 종목 페이지에서도 양방향 링크 (브레인스토밍 Q1 = 3)
- 데이터는 **리포트 ingest 시점 스냅샷**으로만 수집, 별도 스케줄 없음 (Q2 = 1)
- 지표는 **지표별 개별 페이지** (Q3 = 1)
- 각 페이지에 **스냅샷 + 셀사이드 전망 + 영향 매핑**까지 담음 (Q4 = 3)

## 2. 구조 — 새 페이지 타입 `macro`

- 새 디렉토리 `wiki/macro/` 를 기존 `stocks/sectors/analysts/themes` 와 동급으로 추가.
- 지표당 1페이지, 총 7개:

| 파일 | 담는 것 | category | 전망 추적 |
|------|---------|----------|-----------|
| `원달러환율.md` | USD/KRW | 환율 | 예 |
| `유가.md` | WTI · 브렌트 | 원자재 | 예 |
| `VIX.md` | 변동성 지수 | 변동성 | 아니오 |
| `공포탐욕지수.md` | CNN 0~100 | 변동성 | 아니오 |
| `한국금리.md` | 기준금리 · 국고채 3년/10년 | 금리 | 예 |

## 3. 프론트매터 스키마

```yaml
type: macro
name: 원달러환율
category: 환율        # 환율 | 금리 | 원자재 | 변동성 | 신용
unit: KRW/USD         # 지표 단위 (예: USD/bbl, %, bp, pt, 0~100)
created: YYYY-MM-DD
updated: YYYY-MM-DD
```

복수 시리즈를 담는 페이지(예: 한국금리=기준금리+국고채 3년/10년)는 `unit` 에 대표 단위를 쓰고,
스냅샷 히스토리 표에 시리즈별 컬럼을 둔다(§4 참조).

## 4. 본문 섹션 (지표 페이지 공통)

순서 고정:

1. **최신 스냅샷** — 한 줄 요약. 최신값 · 날짜 · 출처. (페이지 상단 빠른 참조)
2. **스냅샷 히스토리** — `날짜 | 값 | 출처` 표. **append-only**, 덮어쓰지 않음.
   - 복수 시리즈 페이지는 값 컬럼을 시리즈별로 분리. 예) `날짜 | 기준금리 | 국고3년 | 국고10년 | 출처`
3. **셀사이드 전망** — `증권사 | 날짜 | 전망레벨 | 기간 | 출처` 표. 환율·유가·금리·CDS 페이지만 둔다.
   VIX·공포탐욕지수 페이지에는 이 섹션을 두지 않는다.
4. **영향 관계** — 지표 변동이 어떤 [[종목]]/[[섹터]]에 어떻게 작용하는지. 위키링크로 연결.
   예) 원화약세 → [[삼성전자]]·[[SK하이닉스]] 수출 수혜 / 유가↑ → 정유·항공 영향.
5. **관련 출처 링크** — `raw/reports/` 또는 외부 출처.

## 5. 데이터 흐름 (리포트 ingest 시점 스냅샷)

INGEST 워크플로에 단계를 추가한다:

1. 리포트를 ingest할 때, 그날(리포트 기준일)의 거시지표 7종을 웹검색으로 수집.
2. 각 macro 페이지의 **스냅샷 히스토리**에 해당 날짜 1행을 append.
3. **Idempotent 보장**: 같은 날짜 행이 이미 있으면 건너뛴다(재실행 안전).
4. `최신 스냅샷` 한 줄과 프론트매터 `updated` 를 갱신.
5. 리포트 주제와 관련된 지표는 해당 종목/섹터 페이지에서 `[[원달러환율]]` 식으로 링크하고,
   거시 페이지의 `영향 관계` 섹션에도 대응 종목/섹터를 링크(양방향).

별도 정기 스케줄(cron) 수집은 이번 범위에 포함하지 않는다.

## 6. 연결 & 인덱스

- `index.md` 에 **## 거시 (wiki/macro/)** 섹션 추가. 표: `지표 | 최신값 | 최근 업데이트`.
- 종목/섹터 페이지 본문에서 관련 거시지표를 위키링크로 양방향 연결.

## 7. CLAUDE.md (스키마 계약) 갱신

- **3계층 구조** 설명에 `wiki/macro/` 추가.
- **페이지 타입** 절에 "거시 페이지 (wiki/macro/)" 항목 추가: 프론트매터 + 본문 섹션 정의(본 문서 §3·§4).
- **INGEST 워크플로**에 거시 스냅샷 수집 단계 추가(본 문서 §5).
- 기존 원칙(raw 불변 / 정보 비삭제 / 수치는 출처·날짜 동반 / idempotent)은 그대로 적용.

## 8. 범위 밖 (YAGNI)

- 정기 자동 수집(cron/스케줄러).
- 차트·그래프 렌더링.
- 거시 대시보드 단일 페이지(개별 페이지 방식 채택으로 불필요).
- 지표 간 상관/회귀 분석.

## 9. 완료 기준

- `wiki/macro/` 에 7개 지표 페이지가 §3·§4 구조로 존재.
- `index.md` 에 거시 섹션 존재.
- `CLAUDE.md` 에 macro 페이지 타입과 INGEST 단계가 반영.
- 다음 리포트 ingest 시 거시 스냅샷이 append되고 재실행해도 중복 행이 생기지 않음.
